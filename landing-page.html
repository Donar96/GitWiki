Einführung

Git ist ein verteiltes Versionskontrollsystem, das von Linus Torvalds entwickelt wurde. Es ermöglicht Teams, effizient und koordiniert an Softwareprojekten zu arbeiten.

Wichtige Befehle

  • git init: Ein neues Repository initialisieren.
  • git clone [URL]: Ein bestehendes Repository klonen.
  • git add [Datei]: Änderungen zur Staging-Area hinzufügen.
  • git commit -m "Nachricht": Änderungen committen.

Branch-Management

Mit Branches können verschiedene Entwicklungsstränge getrennt bearbeitet werden. Wichtige Befehle:

  • git branch: Liste der Branches anzeigen.
  • git checkout [Branch-Name]: Zu einem Branch wechseln.
  • git merge [Branch-Name]: Änderungen aus einem Branch mergen.

Troubleshooting

Häufige Probleme und Lösungen:

  • git status: Status des Repositories überprüfen.
  • git log: Commit-Historie anzeigen.
  • git reset: Änderungen zurücksetzen.
